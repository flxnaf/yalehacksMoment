How it works

Three layers of perception,
always running.

Every camera frame passes through a pipeline that combines machine learning depth, computer vision, and spatial audio to build a continuous picture of the environment.

01

Camera Input

Live frames arrive at up to 30fps from either the iPhone camera or Meta Ray-Ban glasses via the MWDAT SDK and WebRTC stream.

→
02

Depth Inference

DepthAnything V2 (CoreML, ANE-accelerated) produces a 448×448 monocular depth map in ~43ms per frame — 0 = far, 1 = near.

→
03

Spatial Audio

The depth map feeds the SpatialAudioEngine — obstacle pings, a 3D shrine beacon, and Gemini voice all play through HRTF spatial audio.

→
04

Gemini Live

A persistent WebSocket to Gemini 2.5 Flash handles voice conversation, obstacle descriptions, navigation commands, and tool calls — all in real time.

Depth Perception

Seeing distance
in real time.

DepthAnything V2 runs entirely on-device — no cloud, no latency. Each frame is resized to 448×448, passed through the model on Apple's Neural Engine, and produces a full depth map in under 50ms.

The raw float map is normalized, colorized with a perceptual LUT using Accelerate's vDSP, and rendered as an overlay the user or caregiver can see. The same raw values drive all audio decisions.

Spatial Audio

A 3D soundscape
around you.

The SpatialAudioEngine places every sound in a precise direction using HRTF (Head-Related Transfer Function) — the same technique used in gaming to simulate 3D audio over headphones.

The shrine beacon is anchored to a real GPS coordinate and stays fixed in space as you rotate and walk. Gyro-relative tracking ensures "12 o'clock" always means straight ahead, not north.

SpatialAudioEngine.swift 
let env = AVAudioEnvironmentNode()
env.listenerPosition = AVAudio3DPoint(
  x: 0, y: 0, z: 0
)
env.reverbParameters.enable = true

// Place beacon at GPS-anchored bearing
beacon.position = AVAudio3DPoint(
  x: sin(bearingRad) * dist,
  y: 0,
  z: -cos(bearingRad) * dist
)
